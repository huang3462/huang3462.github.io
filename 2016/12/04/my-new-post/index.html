hexo+hithub的血泪史





NO investing,No harvesting

经过几天的奋斗，这尼玛终于将hithub+hexo 的搭建摸了个遍，虽然还有很多地方没有搞清楚，但是还是写自己的第一篇blog，尽管还没有属于自己的域名，但后面会有的啦，哈哈哈哈哈哈哈哈。首先得感谢嘟嘟MD这货。

虽然吧感觉这货好不到哪去（开个玩笑）但是还是他教了我许多，不然我感觉现在我还在坑里不能自拔。

来吧，入坑开始

一开始想到用github搭博客有几个原因： 弄域名和服务器比较麻烦，其次在某个视屏网站看到了github，决定弄个试试：因为如果有个自己的域名平时没事写写东西，也挺不错的，最重要的是你能看到很多大牛的东西，陌陌搭。

不瞎鸡巴扯了，转入正题。

扯个题外话，搞这种东西你必须得学会找资料，简单的说你必须得有个好教程先带着你做一点，一定要那种完整的，别东搞一点，西搞一点，这样是做尼玛的皮皮虾哦，（我之前就是这样的，很烦，感觉被带入屎坑一样，之后我会详细写一点那个屎坑的环境，幸好我舍友及时将我从屎坑里拉了出来） 哈哈哈哈哈哈，“得一好舍友，足矣笑天下”

这人疯了

恩哼， 首先把，我们一进去肯定是得先登入那个叫做github的网站吧，是的，就是它，那一进去一脸懵逼的网站，“ 握草什么鬼，尼玛还全英文”，心里估计有一万个草拟吗飞驰而过了，“劳资4级没过你爷爷的叫我看这个”，“omg”之类的，但是一定别慌呀，别看到全英文的就瞎几把放弃了，俗英说的好“no investing，noharvesting”，

博主 我想删你两耳光子

哈哈，所以嘛，虽然都是全英文的，但是他奶奶的信阳人都放弃了（所以说嘛败不比信阳人）是得加油哦，其实也没那么难懂，真的，没骗人，几个英文单词而已，通俗易懂的嘛（哇哈哈哈哈哈）。

恩你已经成功一半了，看到这

是吧，你已经进去了，并成功创建了自己的账号，但是创账号的时候千万得记住，那个邮箱不能瞎几把乱填哦，你以后修改密码或者其他一系列的操作还得靠你这邮箱的呢（我第一个号的就忘了 zz很鱼）

恩 不错，fighting

账号创建好之后，后面有什么start a project的什么一系列类似于教程一样的东西，可看可不看吧，最好有能力的还是得看看。

又想打人了

别呀，得淡定，冲动是魔鬼，好啦，前面扯得瞎几把多了，这才是正轨的开始，搭博客，得有什么呢，

这不废话嘛

肯定得有空间啦，不然放哪呢？

机智

是的，首先得创建一个空间，那么空间如何创呢？登陆hithub，右上方有个repositories，是的，别管他怎么读，就是这个家伙，点进去，只改一个东西！！！！！ 也是最重要的，那就是你的项目名，一定是“xxx.github.io”xxx是你的账号名，得记住，不然是要出大事的哦。

reporsitories创建完之后，我们就相当于有了自己的一个仓库，仓库干嘛的，这不废话嘛，肯定用来存空间嘛，我们就有了一个空间。

在进行下一步之前，需要下载两个东西，nodejs和hit客户端，那么这两个到底用来作甚呢？

nodejs是什么呢？ 因为搭建这个我们需要用到hexo（一种静态的博客框架）

又是到了说题外话的时候了

在一开始的时候，我就掉进了一种无底洞中，之前我所看到的教程，基本都是在自己电脑创建一个分支，相当于用这个分支自己创建一个页面，简单说就是什么都自己瞎搞，什么配置文件那，都得自己来，这样比较繁琐，这里还是推荐用hexo来设计（还可以用jekyll但在下认为前者更简便一些,）。

言归正传，hexo就是一种静态博客框架，这什么意思呢？就是已经有人写好了那些配置文件，等等，还帮你搞了个主题（之后对主题不满意的话可以更换，具体细节之后再写），你只需要在这个主题中搞一些其他东西就行了。

git客户端就相当于你在自己电脑装个客户端，当你想上传blog的时候用客户端就行了，此外一些ssh秘钥和配置也需要在这个客户端上进行。

啊，终于要开始繁琐的配置之旅了

其实也不多，就主要分为几大部而已。

准备拿刀片了

zz 是的，主要就分为

一 安装Hexo

二 部署本地文件到github中

三 配置ssh秘钥

四 验证

好啦，磨刀不误砍柴公，默默听听我一一道来

当你下载并安装好nodejs后（记住得安装安装安装安装！重要的事得说三遍，我说了4遍了，你懂得）就输入一下命令：

npm install -g hexo

hexo init

npm install

输入完这些之后你就装了个漂亮的主题了，不过还不能再那个丑域名上运行，不过你可以在localhost：4000 上先预览一下

接下来就是部署本地文件到github中了，这个是什么意思呢？简而言之就是将你的这个主题（下载的hexo部署到hithub）中，恩，就是这样的。怎么部署呢？只要打开config最后面加上deploy:

type: git

repository: https://github.com/huang3462/huang3462.github.io.git

branch: master 这样就行了。

接下来就是配置ssh秘钥了，具体步骤我就不写了。

我的天，太懒了吧

这个其实真没必要，基本都是一些命令，最后在自己的hithub中稍作修改就行了，网上教程基本一致，默默搭。

哇哈哈哈哈哈，终于临近结尾了

恩，如你所愿，接下来就是测试了，输入hexo-g，hexo-d，在登陆自己的域名上就能看到一个摸摸摸摸搭的主页了。恩，就是这样。

一个阶段的结束，意味着另一个阶段的开始。

是的，这不是终点，这只是一个起点，人生永远没有终点，有的只是你一直拼搏留下的起点。之后还有很多需要学习的，上传blog，md的学习使用，页面的精修等等等等，“Life not only ，fighting”

稍作总结：多折腾，还是那句我最喜欢的话送给我们：

No，investing，No harvesting

感觉还是有人想扁我。
